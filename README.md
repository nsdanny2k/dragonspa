DragonSpa is a Java-based SPA management system that was successfully deployed and operated in a real-world environment for more than a year.

The application explored an operationally cohesive object and database model originally influenced by procedural workflow concepts from dental clinic systems. The resulting data structure proved adaptable for both SPA and dental operational environments, demonstrating strong flexibility in business workflow modeling.

Originally developed during the Sun Microsystems Java era using traditional JSP/Servlet web application approaches, the project reflects early experimentation in web-based operational systems beyond desktop-oriented business software.

Due to the age of the development environment and dependencies, the project is no longer directly compilable in its original form without modernization effort. However, the underlying architecture and operational workflow concepts remain valuable as a historical foundation for SPA and clinic-oriented management systems.
